01 — Introduction

What is Memory Management?

Memory management is the process of controlling and coordinating a computer's main memory. It involves allocating portions of memory to programs when they request it, and freeing it for reuse when no longer needed. Without proper memory management, a system quickly runs into crashes, memory leaks, and degraded performance.

In this project, we simulate the core memory management operations of an operating system using a singly linked list, where each node represents a block of memory with metadata about whether it is allocated or free.

🧠
Memory Allocation
Assigning free memory blocks to requesting processes using First-Fit strategy.
♻️
Deallocation
Marking used blocks as free and merging adjacent free blocks (coalescing).
🔗
Linked List
Each memory block is a node; pointer links create a dynamic, flexible structure.
⚡
Fragmentation
Internal & external fragmentation — the core problem memory managers must solve.

Why Linked List for Memory Management?

Arrays are fixed in size and require contiguous space. A linked list allows dynamic sizing — nodes can be inserted, removed, and split at any position in O(1) time once the position is known. This makes it ideal for representing memory blocks that constantly change in size and status.

Memory Block Node Structure

Each node in our linked list stores the following information about a memory block:

typedef struct MemoryBlock {
    int   size;          // size of this block in KB
    int   process_id;   // -1 if free, else PID of owner process
    int   start_addr;   // starting address in simulated memory
    int   is_free;      // 1 = free, 0 = allocated

/*

*/

Allocation Strategies — Comparison

StrategyHow It WorksSpeedFragmentation
First-Fit Scan from head, take the first block ≥ requested size ⚡ Fastest Medium
Best-Fit Scan all, pick the smallest block that fits 🐢 Slow Lowest
Worst-Fit Scan all, pick the largest free block 🐢 Slow Highest
Our project implements First-Fit — the most practical strategy used in real OS kernels like Linux's older allocators, because it is O(n) in worst case but extremely fast in typical usage.
03 — Implementation

Complete C Implementation

Below is the full, compilable C source code with detailed inline comments explaining every operation. Copy this code into a .c file and compile with gcc memory_sim.c -o memory_sim.

memory_sim.c — Part 1: Headers & Structure 
/*=============================================================
  MEMORY MANAGEMENT SIMULATOR USING LINKED LIST
  Subject  : Data Structures & Algorithms
  Language : C
  Strategy : First-Fit Allocation
=============================================================*/

#include <stdio.h>
#include <stdlib.h>
#include <string.h>

#define TOTAL_MEMORY  1024   /* Total simulated RAM: 1024 KB */
#define FREE          1
#define ALLOCATED     0

/* ─── Node: one memory block in the linked list ─── */
typedef struct Block {
    int  size;          /* block size in KB                  */
    int  pid;           /* process ID, -1 if free            */
    int  start;         /* starting address (simulated)      */
    int  is_free;       /* FREE(1) or ALLOCATED(0)           */
    struct Block *next; /* pointer to next block             */
} Block;

/* Global head pointer — start of our memory list */
Block *head = NULL;
memory_sim.c — Part 2: Initialize Memory 
/*─────────────────────────────────────────────────────────
  initialize_memory()
  Creates a single node representing all memory as FREE.
  This is the starting state before any allocations.
─────────────────────────────────────────────────────────*/
void initialize_memory() {
    head = (Block*) malloc(sizeof(Block));
    if (!head) {
        printf("Fatal: malloc failed\n");
        exit(1);
    }
    head->size    = TOTAL_MEMORY;  /* entire memory = 1024 KB */
    head->pid     = -1;           /* no owner                */
    head->start   = 0;            /* starts at address 0     */
    head->is_free = FREE;
    head->next    = NULL;
    printf("[INIT] Memory initialized: %d KB free.\n", TOTAL_MEMORY);
}

/*─────────────────────────────────────────────────────────
  create_block() — Helper: creates a new Block node
─────────────────────────────────────────────────────────*/
Block* create_block(int size, int start, int pid, int is_free) {
    Block* b = (Block*) malloc(sizeof(Block));
    b->size    = size;
    b->start   = start;
    b->pid     = pid;
    b->is_free = is_free;
    b->next    = NULL;
    return b;
}
memory_sim.c — Part 3: Allocate Memory (First-Fit) 
/*─────────────────────────────────────────────────────────
  allocate_memory(pid, size)
  First-Fit Strategy:
    1. Traverse from head.
    2. Find the first FREE block with size >= requested.
    3. If block is larger, SPLIT it into:
         [allocated block] → [remaining free block]
    4. Mark the block as ALLOCATED and assign PID.

  Time Complexity: O(n) — must traverse up to n nodes
─────────────────────────────────────────────────────────*/
void allocate_memory(int pid, int size) {
    Block* curr = head;

    while (curr != NULL) {
        /* Check: is this block free AND big enough? */
        if (curr->is_free == FREE && curr->size >= size) {

            /* ── CASE 1: Block is larger — split it ── */
            if (curr->size > size) {
                /* Create the leftover free fragment */
                Block* frag = create_block(
                    curr->size - size,          /* remaining size   */
                    curr->start + size,          /* new start addr   */
                    -1, FREE                     /* free, no PID     */
                );
                frag->next  = curr->next; /* fragment → rest of list */
                curr->next  = frag;      /* curr → fragment         */
            }

            /* ── CASE 2 or after split: mark as allocated ── */
            curr->size    = size;
            curr->pid     = pid;
            curr->is_free = ALLOCATED;

            printf("[ALLOC] PID %d: Allocated %d KB @ addr %d\n",
                    pid, size, curr->start);
            return;
        }
        curr = curr->next; /* move to next block */
    }
    /* No block found — allocation fails */
    printf("[FAIL] Not enough memory for PID %d (%d KB)\n", pid, size);
}
memory_sim.c — Part 4: Free Memory + Coalescing 
/*─────────────────────────────────────────────────────────
  free_memory(pid)
  Steps:
    1. Find the block owned by pid.
    2. Mark it free (pid = -1).
    3. COALESCE: merge with the next block if it's also free.
    4. (For full coalescing, a doubly linked list is ideal;
       here we handle forward merging for simplicity.)

  Time Complexity: O(n)
─────────────────────────────────────────────────────────*/
void free_memory(int pid) {
    Block* curr = head;

    while (curr != NULL) {
        if (curr->pid == pid && curr->is_free == ALLOCATED) {

            curr->is_free = FREE;  /* mark free */
            curr->pid     = -1;

            printf("[FREE] PID %d released %d KB @ addr %d\n",
                    pid, curr->size, curr->start);

            /* ── COALESCE with next block if also free ── */
            if (curr->next != NULL && curr->next->is_free == FREE) {
                Block* temp        = curr->next;
                curr->size        += temp->size; /* absorb next block */
                curr->next        = temp->next;
                free(temp);                       /* free the merged node */
                printf("[COAL] Merged adjacent free blocks.\n");
            }
            return;
        }
        curr = curr->next;
    }
    printf("[FAIL] PID %d not found or already free.\n", pid);
}
memory_sim.c — Part 5: Display & main() 
/*─────────────────────────────────────────────────────────
  display_memory()
  Traverses the linked list and prints every block.
─────────────────────────────────────────────────────────*/
void display_memory() {
    Block* curr = head;
    int total_free = 0, total_used = 0, block_num = 1;

    printf("\n══════════════════════════════════════════════\n");
    printf("  MEMORY MAP  (Total: %d KB)\n", TOTAL_MEMORY);
    printf("══════════════════════════════════════════════\n");
    printf("  %-6s %-10s %-10s %-10s %-8s\n",
           "Blk", "Start", "Size(KB)", "Status", "PID");
    printf("  ─────────────────────────────────────────\n");

    while (curr != NULL) {
        printf("  %-6d %-10d %-10d %-10s %-8s\n",
            block_num++,
            curr->start,
            curr->size,
            curr->is_free ? "FREE" : "USED",
            curr->is_free ? "---"  : /* pid as string via sprintf */
                (char[10]){}/* snprintf into tmp buf */
        );
        if (curr->is_free) total_free += curr->size;
        else               total_used += curr->size;
        curr = curr->next;
    }
    printf("  ─────────────────────────────────────────\n");
    printf("  Free: %d KB   Used: %d KB\n\n", total_free, total_used);
}

/*─────────────────────────────────────────────────────────
  main() — Demo run
─────────────────────────────────────────────────────────*/
int main() {
    initialize_memory();           /* 1024 KB all free              */
    display_memory();

    allocate_memory(1, 200);       /* PID 1 gets 200 KB             */
    allocate_memory(2, 300);       /* PID 2 gets 300 KB             */
    allocate_memory(3, 150);       /* PID 3 gets 150 KB             */
    display_memory();

    free_memory(2);               /* PID 2 releases its 300 KB    */
    display_memory();

    allocate_memory(4, 100);       /* PID 4 fits in freed space     */
    display_memory();

    free_memory(1);               /* PID 1 frees → coalesce check */
    display_memory();

    return 0;
}
04 — Algorithms

Step-by-Step Algorithm Walkthroughs

Algorithm 1 — Memory Allocation (First-Fit)

1
Set curr = head (start from the first block)
2
While curr != NULL, check: is it FREE and curr→size ≥ requested?
3
If YES and curr→size > requested: SPLIT — create a new FREE fragment node with the leftover bytes, insert it after curr
4
Set curr→size = requested, curr→pid = pid, curr→is_free = ALLOCATED
5
If NO match found after full traversal → print Allocation Failed (out of memory)

Algorithm 2 — Memory Deallocation + Coalescing

1
Traverse list to find the block with pid == target_pid
2
Set curr→is_free = FREE, curr→pid = -1
3
Coalesce forward: if curr→next != NULL and curr→next→is_free == FREE: merge — add next's size to curr, skip next node, free() it
4
Continue until no adjacent free nodes remain
💡 Pro Tip — Backward Coalescing: To also merge with the previous block, convert to a doubly linked list so you can traverse backwards. This is how the Linux kernel's buddy allocator works.

Algorithm 3 — Compaction

Compaction eliminates external fragmentation by rearranging all allocated blocks to be contiguous, leaving all free space as one large block at the end.

compaction.c 
/*─────────────────────────────────────────────────────────
  compact_memory()
  Rebuilds the linked list by:
    1. Collecting all ALLOCATED blocks (in order).
    2. Assigning new contiguous start addresses.
    3. Adding one large FREE block at the end for
       all the formerly fragmented free space.
─────────────────────────────────────────────────────────*/
void compact_memory() {
    Block* curr       = head;
    int  next_addr   = 0;
    int  total_free  = 0;

    /* Pass 1: repack allocated blocks, collect free total */
    while (curr != NULL) {
        if (!curr->is_free) {
            curr->start = next_addr;     /* assign new address */
            next_addr  += curr->size;
        } else {
            total_free += curr->size;    /* accumulate free    */
        }
        curr = curr->next;
    }

    /* Pass 2: remove all free nodes */
    Block** pp = &head;
    while (*pp) {
        if ((*pp)->is_free) {
            Block* tmp = *pp;
            *pp = tmp->next;
            free(tmp);
        } else {
            pp = &(*pp)->next;
        }
    }

    /* Append one big FREE block at the end */
    Block* free_blk = create_block(total_free, next_addr, -1, FREE);
    *pp = free_blk;

    printf("[COMPACT] Done. %d KB free space consolidated @ addr %d\n",
            total_free, next_addr);
}

Complexity & Performance Analysis

Allocate O(1) O(n) O(1)* Best if head is free; worst = full scan
Free O(1) O(n) O(1) O(1) if we store a direct pointer
Coalesce O(1) O(n) O(1) O(1) forward merge; O(n) backward

Display O(n) O(n) O(1) Must visit all nodes

* O(1) extra space — we reuse the existing block; a split adds exactly 1 new node.

Fragmentation Analysis

🧩
External Fragmentation
Many small free blocks exist but no single block is large enough. Solved by compaction or coalescing.
📦
Internal Fragmentation
Allocated block is larger than needed. We minimize this by splitting blocks precisely during allocation.
✅ Our implementation minimizes internal fragmentation by splitting blocks to exactly the requested size, leaving a properly tracked free fragment. Coalescing after each free() reduces external fragmentation.

08 — Real-World Context

Real-World Applications

🐧
Linux Kernel
The Linux slab allocator and buddy system use linked lists of free blocks grouped by power-of-2 sizes for O(log n) allocation.
☕
Java GC
Java's garbage collector maintains free lists (linked lists) to track reclaimed heap memory between GC cycles.
🦀
Rust Allocator
Rust's global allocator uses linked free-lists with size classes to achieve near-zero overhead memory management.
📱
Embedded Systems
Microcontrollers with no OS use custom linked-list allocators (like FreeRTOS heap_4) to manage limited RAM.

Conclusion

This project demonstrates how a seemingly simple data structure — the singly linked list — forms the backbone of one of the most critical operations in computing: memory management. The key takeaways are:

✅ Dynamic Flexibility
Linked lists allow blocks to be created, split, merged, and destroyed at runtime without any fixed structure.
✅ Pointer Power
Pointer manipulation enables O(1) block splitting and merging — something impossible with arrays.
✅ Real OS Concepts
First-Fit, coalescing, and compaction are actual techniques used in production operating systems.
